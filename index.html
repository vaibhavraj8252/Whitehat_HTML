What I learnt on Whitehat Jr.?

C1 (Object Oriented Programming)

Orient students to think about programming in an object-oriented manner. Student designs Paddle and Ball class to create objects for use it in their program.

C2 (Sprite Objects)

Students use Sprite class to create an animated ball in a playground project where they experiment with different sprite properties and functions

C3 (Conditional Programming)

Student uses conditional programming to add control to the ball's movements. Student builds a little game using the ball's movements and adds some challenge to it.

C4 (Design Pong Game)

Students use knowledge of sprites, object properties and functions to create and assign game behavior to the objects in the Pong Game

C5 (Functions and Loops)

Student learns to create abstractions in their code by declaring functions. Student also learns to replace repetitive code with a for-loop.

C6 (Game State and Two Player Game)

Student learns to store the state of a game in a variable. Student assigns different behaviour to the objects in the game depending on the state of the game.

C7 (Animation and Sound)

Students learn to add sound effects and animation effects to the game.

C8 (The World's Hardest Game)

Students review the concepts taught in Classes 1 to 7 while building the “The World’s Hardest Game”.

C9 (T-Rex Runner)

Students learn to create a dinosaur with jump and run animations and indent code to make it more readable.

C10 (Infinite Game World)

Students learn to create an infinite game world in the limited screen space available to the players.

C11 (Random Numbers)

Students learn how to spawn game objects at random positions in the game.

C12 (Memory Leak and Switch Statements)

Students learn about memory leaks and use switch statements in their code to randomly spawn different varieties of obstacles. Students also build a simple scoring system.

C13 (Game States and Groups)

Students learn to assign different game behavior for different states in the game. Students also learn to add similar objects in a group and assign the same behavior to all the group objects.

C14 (Bugs - The Curious Case of Disappearing Game Objects)

Students solve the problem of disappearing game objects inside the game. Students also learn about the collider radius and add an animation to display the game end state and write a function to reset the game.

C15 (Game Adaptivity)

Student adds sounds to the game. Student learns to make the game increasingly challenging for the player as the game progresses. Student also learns to add intelligence to the Trex so that it jumps automatically on seeing an obstacle..

C16 (Scope of Variables)

Students will learn about the global and local scope of variables. Students will write a reset function for the game.

C17 (Game Adaptivity)

Student adds sounds to the game. Student learns to make the game increasingly challenging for the player as the game progresses. Student also learns to add intelligence to the Trex so that it jumps automatically on seeing an obstacle.

C18 (Publishing an App on Play Store)

Students will learn to adjust their game dimensions to fit all screen sizes. Students will then build a mobile wrapper around the application to generate an apk file that can be published on playstore.

C19 (Super Mario)

Students upload the Super Mario game on the 200 OK! Web Server and host it locally.

C20 (Designing Algorithms)

Students learn to write code on a standard code editor (Visual Studio Code). Students design algorithms for collision detection and bounce-off. Students learn how to pass arguments in a function. They’ll also learn to run and test their code on their local system.

C21 (Writing functions which can take argumanets)

Student learns to write a function which can accept arguments , return values and can be reused for different game objects. Student also learns to create their own little code library.

C22 (Physics Engine)

Students will be able to use a Physics engine (Matter.js) and create a world with objects in it using the engine.

C23 (Creating Blueprints)

Students will learn how to create a class in JavaScript and use it to create new objects using the physics engine. Students will make two box objects using a class that falls to the ground and then topples over each other.

C24 (Classes and Objects)

Students will design the different classes corresponding to the different game objects in the game - the pig objects, the box objects, the logs, the ground and the bird. Student will create the different objects using the above classes and stack them on the screen. Student will also make the bird object move with the mouse so that it can interact with the other objects.

C25 (Inheritance)

Student will add images and animations to the box model angry birds game which they developed in the last class. Student will also modify the code to use inheritance to write sub-classes which inherit properties and functions from their parent class. Student will connect this to the DRY principle studied in the earlier classes.

C26 (Git and Github)

Students will learn how to use different git commands to set up a repository, clone a repository, commit changes etc. remotely through command line

C27 (Constrained Body)

Student will learn about constrained bodies and how to create constrained bodies using matter.js. Student will also create two constrained bodies using Matter.Constraint

C28 (Slingshot)

Students create a slingshot effect in their Angry Birds game.Students also learn to employ the mouse events like mouseDragged and mouseReleased

C29 (Catapult and Rubber band)

Students will add the catapult and the rubber band in the game.

C30 (Vanishing Pigs)

Students learn to manipulate a property of a class to give a vanishing effect to pig object under certain conditions. Students also learn keyboard events and ASCII values of the keys.

C31 (Arrays and Bird Trajectory)

Student learns about the different ways in which data can be stored in javascript. Student explores the array data structure to design the bird's trajectory after it has been launched.

C32 (JSON and API calls)

Student will learn about the JSON data structure which can be used to store and access data. Students will learn what is an API and how an API call be made to the service and desired data extracted from it. Students will use an API call to change the background of the game depending on the time of the day at a specified place. Student will also build the scoring system in the game.

C33 (Scoring and Debugging)

Students also learn how to debug the errors in the Angry Bird game. Students will also build the scoring system in the game.

C34 (Crashing Tram)

Students review the concepts taught in Classes 23 to 33 while building the Angry Birds Game.

C35 (Realtime Database)

Students learn the importance of using a Realtime database to create multiplayer games. Students learn how to connect, read and write data into a remote real time database.

C36 (Structuring before Coding)

Students design a form using p5 dom to allow players to login with their name. Students use the OOPs programming style to write the code.

C37 (Game players registration in the Database)

Students create a form to register the player’s name. Students also create a gameState to resist the number of users to be registered and start the game as soon as it reaches the count.

C38 (Game in the Database)

Students create a layout for a multiplayer car racing game in the database. Students press the up arrow key to change the distance covered property of each player in the database. The distances covered by all players are shown on the screen with the player playing the game highlighted.

C39 (Game Camera and Display Size)

Students use a database to design the car racing game when the game is in play and use a game camera to focus on the game's active player. Students also adjust the game canvas to the size of the display device.

C40 (Real Car Racing)

Students will replace the sprites with real cars of their choice. Students will add a racing track as the background. Students will also write a condition to end the game. Students will also create a reset button at the top of the game to reset the playerCount and gameState in our game.

C41 (Changing Lanes)

Students will add the condition to move the players to left and right when the arrow key is pressed. Students will also add some identification to help the players identify which is their car.
